description: 해커톤·신규 기능 아이디어를 받아 MVP Spec 문서로 구체화한다. 문제·사용자·핵심기능·제외범위·성공기준·기술스택을 정리할 때 사용.
---

아이디어를 받으면 아래 항목을 차례로 묻고, 빠진 항목은 되물어 MVP Spec을 완성한다.

1. 문제 — 누구의 어떤 불편을 푸는가
2. 사용자 — 핵심 사용자 (페르소나)
3. 핵심 기능(MVP) — N일에 만들 수 있는 M개로 제한
4. 제외 범위 — 이번엔 안 하는 것 (중요)
5. 성공 기준 — 시연에서 무엇이 되면 성공인가
6. 기술 스택 — 팀이 익숙한 것 우선

결과는 spec.md 형식의 마크다운으로 출력한다.
범위가 크면 반드시 줄이도록 제안한다.
