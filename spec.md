# TaskPilot MVP Spec

## 1. 개요

**프로젝트명:** TaskPilot  
**슬로건:** AI 기반 우선순위 자동 관리 To-do 앱  
**개발 기간:** 30일 (1명)  
**배포 대상:** 웹 (반응형 UI, PC/모바일 지원)

---

## 2. 문제

### 현재의 불편함
- **업무 다중 채널:** Slack, 이메일, 메시지 등 다양한 채널에서 업무 요청이 산발적으로 옴
- **우선순위 관리 어려움:** 들어오는 일들의 중요도와 긴급도를 빠르게 판단하기 어려움
- **일정 관리 시간 소비:** 매일 들어오는 일들을 수동으로 정리하고 우선순위를 정하는 데 시간 소비

### 해결 목표
- 텍스트 입력만으로 자동 일정 등록
- AI가 우선순위를 판단해 To-do 리스트 제공
- 난이도에 맞춘 실시간 알림으로 업무 진행 지원

---

## 3. 사용자 (페르소나)

**핵심 타겟:** 모든 사람 (General audience)
- 업무/학업/개인 프로젝트 관리가 필요한 모든 사용자

**개인화 전략:**
- 초기 온보딩: 사용자의 업무 특성, 선호 시간, 집중력 패턴 등에 대한 간단한 질문
- 지속적 학습: 사용자의 완료 패턴, 선호 우선순위 패턴 학습해 AI 모델 개인화

---

## 4. MVP 기능 (30일 안에 구현)

### Phase 1: Core (필수)

#### 1.1 자동 일정 등록
- **기능:** 텍스트 입력/붙여넣기 → 자동으로 일정 추출 및 등록
- **입력 예시:** "3월 15일까지 보고서 제출 (2시간 소요)", "내일 오전 10시 회의"
- **처리:** NLU 기반 텍스트 분석으로 제목, 마감일, 예상 소요 시간 추출
- **저장:** 추출된 정보를 DB에 저장

#### 1.2 AI 우선순위 결정 및 To-do 리스트
- **기능:** 등록된 모든 일정을 AI가 분석해 우선순위 점수 계산
- **우선순위 판단 기준:**
  - 마감일까지 남은 시간
  - 예상 소요 시간
  - 사용자의 개인화 선호도 (온보딩 + 학습)
  - 난이도 (사용자 입력 또는 추론)
- **화면:** 우선순위 순서로 정렬된 To-do 리스트 제공 (실시간 업데이트)

#### 1.3 난이도별 알림 (카카오톡)
- **기능:** 태스크의 난이도와 남은 시간을 고려해 최적 타이밍에 카카오톡 알림 전송
- **알림 로직:**
  - 쉬운 일: 마감 1일 전
  - 중간: 마감 2일 전
  - 어려운 일: 마감 3-4일 전
- **알림 내용:** "TaskName (난이도: 중간, 예상: 2시간) 시작하기 좋은 시간입니다"

#### 1.4 대시보드 & 반응형 UI
- **대시보드 구성:**
  - 오늘의 우선순위 To-do (상단)
  - 전체 일정 타임라인 (주간/월간 뷰)
  - 완료율, 평균 완료 시간 등 간단한 통계
  - 사용자 설정 (개인화 선호도 조정)
- **반응형:** PC(1200px 이상) & 모바일(320px 이상) 모두 최적화

## 5. 제외 범위 (v1.0에서는 안 함)

- ❌ 팀 공유 기능 (개인용 전용)
- ❌ 반복 일정 (일회성만)
- ❌ 채널 통합 (Slack, Gmail 등 외부 API 연동은 v2.0)
- ❌ 네이티브 모바일 앱 (웹 반응형만)
- ❌ 음성 입력
- ❌ 소셜 공유

---

## 6. 성공 기준 (데모에서 확인)

### MVP 인정 체크리스트
- [ ] 텍스트 입력 → 자동 일정 등록 및 DB 저장
- [ ] AI가 우선순위 계산 → To-do 리스트 표시
- [ ] 우선순위가 실제 마감일/소요시간 기반으로 변경되는 것 확인
- [ ] 카카오톡으로 최소 1회 이상 알림 수신 확인
- [ ] 모바일에서 대시보드 반응형 UI 확인 (터치 가능)
- [ ] 온보딩 후 사용자 선호도가 우선순위에 영향 주는 것 확인 (정성적)

## 7. 기술 스택

| 계층 | 기술 | 비고 |
|------|------|------|
| **Frontend** | Next.js 16 + React 19 + TypeScript | 반응형 UI (Tailwind CSS 또는 CSS Modules) |
| **Backend** | Spring Boot 3.x | REST API, Spring Data JPA |
| **Database** | PostgreSQL 15+ | 일정, 사용자, 알림 로그 저장 |
| **NLU (텍스트 분석)** | OpenAI GPT API 또는 한국어 NLU 라이브러리 | 마감일/시간 추출 |
| **AI (우선순위)** | OpenAI GPT API 또는 간단한 점수 알고리즘 | 규칙 기반 또는 LLM 기반 |
| **알림** | 카카오톡 API (카카오 비즈니스) | 사용자 인증 연동 |
| **배포** | Vercel (Next.js) + AWS/GCP (Spring Boot) | 또는 Docker 컨테이너화 |

## 9. 주의사항 & 리스크

### ⚠️ 리스크
1. **Spring Boot 첫 경험:** 서버 구축 곡선이 가파를 수 있음
   - **완화:** Spring Boot 튜토리얼 선행, 복잡한 기능은 v2.0으로 미룰 것
2. **NLU/AI 구현의 복잡성:** 한국어 텍스트 분석이 까다로울 수 있음
   - **완화:** OpenAI API 활용 (비용 발생), 또는 간단한 정규식 기반 추출로 시작
3. **30일 타이트한 일정:**
   - **완화:** MVP는 작게 유지, 핵심 기능에 집중

### ✅ 성공 포인트
- 개인용 앱이라 권한/공유 로직 단순
- 웹만 지원하므로 플랫폼 관리 복잡도 낮음
- 초기엔 "우선순위 점수 알고리즘"을 간단하게 시작해 점진적으로 개선 가능
